Introduction

This project proposes a generalized framework for recognizing and understanding expiration date of the products. The proposed framework can handle challenging expiration date cases and distinguish 13 different date formats shown in Table 1. Moreover, it can accurately detect and recognize expiration dates even when the input image contains multiple dates. It generates bounding boxes with predictions in the image. Figure 1 shows the results of recognition and understanding expiration date.

Executable File

We've released executable files of the proposed framework for Windows (run_windows.exe) and Ubuntu (run_ubuntu.exe) operating systems. The executable file for Windows is able to use only cpu whereas the executable file for Ubuntu is able to use both cpu and gpu. Anyone who would like to test their own images for recognition and understanding expiration date can use our executable file. Before running the executable file, first, create a folder called images, that contains test images. Then, place the images folder and the .exe file in the same directory. After running the executable file, a folder called results will be created in the same directory as exe file. Then, the proposed framework will save the predictions in the results folder, by running the following commands; 

# For windows
cd path/to/exe
run_windows.exe
# For ubuntu
cd path/to/exe
./run_ubuntu


ExpDate: Expiration Date Dataset

In the lack of a publicly available dataset, we create a novel dataset, ExpDate, for expiration date detection and recognition. It contains real-world expiration date images captured from food, beverage, and medicines. It also includes challenging cases with various date scales, different date fonts, reflection, complex backgrounds, etc. Figure 2 visualizes some images in ExpDate.



Details

ExpDate contains a total of 1767 images. We split it into training and test sets consisting of 1102 training and 665 test images, respectively. Additionally, the test set contains 1956 day, month, and year components, unlike the training set. ExpDate has near-horizontal expiration dates in the images. The width and height of the images vary in the range of [482, 2048] and [452, 2298] pixels, respectively. The dates are between the years 2018 and 2026. ExpDate has various types of expiration date fonts, including the dot-matrix font. It also includes images with multiple dates on the products (i.e., expiration and production dates). ExpDate includes 13 different expiration date formats shown in Table 1.

Annotation

Each date in the images is annotated with class, bounding box coordinates, date transcription, width, and height of the image in the training set. The test set contains the annotations of the day, month, and year components in addition to date annotation, unlike the training set. Each component is annotated with class, bounding box, and transcription. We utilize the top-left and bottom-right corner coordinates of the bounding boxes.
